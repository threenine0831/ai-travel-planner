# 수동 테스트 체크리스트

## 준비

- [ ] `frontend/js/firebase-config.js`에 Firebase Web 설정을 입력했다.
- [ ] `frontend/js/app-config.js`에 FastAPI 서버 주소를 입력했다.
- [ ] 로컬은 `server/local-settings.txt`, Render는 환경변수에 Google AI API 키와 Firebase Admin 값을 입력했다.
- [ ] Firestore Rules와 Indexes를 배포했다.
- [ ] Firebase Authentication에서 이메일/비밀번호 로그인을 활성화했다.

## 인증

- [ ] 회원가입이 동작한다.
- [ ] 닉네임이 Firebase Auth 프로필과 `users/{uid}`에 저장된다.
- [ ] 로그인이 동작한다.
- [ ] 로그아웃이 동작한다.
- [ ] 새로고침 후 로그인 상태가 유지된다.
- [ ] 로그인 사용자의 닉네임이 헤더에 표시된다.
- [ ] 로그인하지 않은 사용자가 내 일정, 일정 수정, AI 생성 화면 접근 시 로그인 화면으로 이동한다.

## AI 일정 생성

- [ ] 여행 제목 필수 검증이 동작한다.
- [ ] 여행지 필수 검증이 동작한다.
- [ ] 잘못된 날짜 형식을 차단한다.
- [ ] 종료일이 시작일보다 빠르면 오류를 표시한다.
- [ ] 14일 초과 여행을 차단한다.
- [ ] 0 이하 예산을 차단한다.
- [ ] 여행 인원 1명 미만 또는 20명 초과를 차단한다.
- [ ] 관심사 미선택을 차단한다.
- [ ] 추가 요청사항 500자 초과를 차단한다.
- [ ] 생성 중 버튼이 비활성화되고 로딩 메시지가 표시된다.
- [ ] AI 오류 발생 시 한국어 오류와 다시 시도 가능한 상태가 표시된다.
- [ ] 정상 생성 후 `result.html`에 날짜별 일정이 표시된다.

## 저장과 내 일정

- [ ] 생성 결과를 비공개로 저장할 수 있다.
- [ ] 생성 결과를 공개로 저장할 수 있다.
- [ ] 저장 성공 후 중복 저장이 방지된다.
- [ ] 내 일정 목록이 최신 수정순으로 표시된다.
- [ ] 비공개 일정은 작성자 본인만 조회된다.
- [ ] 자신의 일정 상세 화면이 표시된다.
- [ ] 자신의 일정 항목을 수정할 수 있다.
- [ ] 일정 항목을 추가할 수 있다.
- [ ] 일정 항목을 삭제할 수 있다.
- [ ] 여행 팁을 추가할 수 있다.
- [ ] 여행 팁을 삭제할 수 있다.
- [ ] 공개 여부를 수정할 수 있다.
- [ ] 삭제 전 확인 창이 표시된다.
- [ ] 자신의 일정을 삭제할 수 있다.

## 커뮤니티

- [ ] 공개 일정 목록이 최신순으로 표시된다.
- [ ] 비공개 일정은 커뮤니티에 표시되지 않는다.
- [ ] 여행지 검색이 동작한다.
- [ ] 더 보기 페이지네이션이 동작한다.
- [ ] 검색 결과 없음 상태가 표시된다.
- [ ] 공개 일정이 없을 때 빈 상태가 표시된다.
- [ ] 다른 사람의 일정에는 수정과 삭제 버튼이 표시되지 않는다.
- [ ] 공개 상세 화면에 이메일과 Firebase UID가 표시되지 않는다.

## 배포와 화면

- [ ] Netlify 배포 주소에서 HTML 페이지 직접 접근이 가능하다.
- [ ] Render `/health`가 정상 응답한다.
- [ ] Netlify 주소가 `ALLOWED_ORIGINS`에 포함되어 CORS 오류가 없다.
- [ ] 모바일 화면에서 가로 스크롤이 없다.
- [ ] 태블릿 화면에서 카드와 폼 레이아웃이 자연스럽다.
- [ ] 데스크톱 화면에서 타임라인과 카드가 깨지지 않는다.
- [ ] 브라우저 콘솔에 명백한 JavaScript 오류가 없다.
- [ ] Render 콜드 스타트 후에도 요청을 다시 시도하면 정상 동작한다.
